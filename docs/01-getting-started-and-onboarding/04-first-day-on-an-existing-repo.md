# 既存リポジトリに参加した初日にやること

## チェックリスト

- `GitHub` / `VS Code` / `Copilot` の初期セットアップが完了していることを確認する
- `README` と開発ルールを読む
- `Issue` 管理の流れを把握する
- `copilot-instructions.md` があるなら読む
- `npm test` / `npm run build` など基本コマンドを確認する
- レビュー文化とブランチ命名規則を確認する

## 最初に意味を把握したいコマンド

- `npm test`
  - `test`: `package.json` に定義されたテスト用 script を実行する
  - **確認したいこと:** 何のテストが動くか、失敗時の見方はどうか
- `npm run build`
  - `run`: 任意の script 名を実行する
  - `build`: build 用 script 名。プロジェクトにより内容は異なる
  - **確認したいこと:** どの成果物が作られるか、公開前に必須か

> 初日は「全部覚える」よりも、**どのコマンドが品質確認の入口か** を把握することを優先すると進めやすいです。

## Copilot への依頼例

- 「この repo の主なディレクトリの役割を説明して」
- 「初日に押さえるべきルールを 5 つにまとめて」
- 「この Issue を始める前提知識を要約して」

## 目標

初日で、**何を変更してよくて、何を勝手に変えてはいけないか** を理解することが重要です。
