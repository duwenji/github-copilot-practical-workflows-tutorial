# ハンズオン: shared repo 戦略を決める

## 目的

`submodule`、`subtree`、コピー運用のどれを採用するか、実務目線で判断できるようにします。

## 推奨題材

- `shared-copilot-skills`
- `github-copilot-custom-agents-tutorial`

## ワーク

1. 共有したい資産を 1 つ決める
2. 更新頻度、利用側の人数、バージョン固定の必要性を整理する
3. `submodule` と `subtree` のどちらが向くか判断する
4. 採用理由を 5 行でまとめる

## 期待するアウトプット

- 判断表
- 採用理由
- 更新手順のたたき台
