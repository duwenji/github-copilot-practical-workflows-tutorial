## 代表シナリオ

1. 既存リポジトリに参加する
2. 日常の機能追加を進める
3. `fork` と `upstream` で本家に追従する
4. `submodule` / `subtree` を使って共有資産を管理する
5. release / hotfix / rollback を回す
6. `copilot-instructions.md` でチーム標準を整える
