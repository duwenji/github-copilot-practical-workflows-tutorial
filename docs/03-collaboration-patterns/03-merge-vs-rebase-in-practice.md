# merge と rebase の実務的な使い分け

## コンセプトと仕組み

- `merge` は分岐した履歴をそのまま統合する考え方です。
- `rebase` は自分の変更を最新の土台に載せ直して、履歴を整理する考え方です。
- つまり、**共有履歴の安全性を優先するか、履歴の見通しを優先するか** で使い分けます。

## ざっくりした違い

- `merge` — 履歴をそのまま残しつつ統合する
- `rebase` — 履歴をきれいに並べ替えて取り込む

## どちらを使うか

- チーム標準があるならそれに従う
- 初学者や共有ブランチでは `merge` の方が安全なことが多い
- 履歴整理を重視するなら `rebase` も有効

## Copilot の使いどころ

- 差分の意味を要約してもらう
- rebase 後に確認すべきポイントを整理する

## 注意点

- 公開済みブランチの `rebase` はチームに影響することがある
- どちらを使っても、最終的にはテスト確認が必要
