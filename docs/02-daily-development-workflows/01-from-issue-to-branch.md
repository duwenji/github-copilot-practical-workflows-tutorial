# Issue から branch を切るまで

## 典型シナリオ

チームの backlog から Issue を受け取り、作業ブランチを作って着手する場面です。

## 基本手順

1. Issue の目的と受入条件を読む
2. 必要なら Copilot Chat で作業分解を相談する
3. ローカルで最新を取り込む
4. 作業ブランチを作成する

```powershell
git pull origin main
git switch -c feature/add-quiz-filter
```

## Copilot の使いどころ

- 「この Issue の作業ステップを 5 つに分けて」
- 「受入条件からテスト観点を整理して」
- 「この変更で影響しそうなファイルを推測して」

## 成果物

- 着手前メモ
- 作業ブランチ
- 変更対象の見通し

## 注意点

- Issue を読まずに実装を始めない
- ブランチ名はチーム規約に合わせる
- 受入条件が曖昧なら先に確認する
