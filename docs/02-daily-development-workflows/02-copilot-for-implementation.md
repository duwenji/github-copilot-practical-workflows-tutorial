# Copilot を使った実装の進め方

## 典型シナリオ

仕様がある程度決まっていて、実装を前に進めたい場面です。

## コンセプトと仕組み

- Copilot はコンテキストをもとに候補を提案するため、ファイル名・関数名・コメントが精度に影響すること
- 提案は必ず確認してから採用することで、意図しないコードの混入を防げること
- 要件・セキュリティ・例外処理は開発者が責任を持って判断すること

## 基本手順

1. 実装したい機能の概要を Copilot Chat に伝えること
2. 提案されたコードの意図とロジックを確認すること
3. テストケースを追加して動作を検証すること
4. 不要な提案は手動で削除または修正すること

## Copilot の使いどころ

- 「この関数にバリデーションを追加してください」
- 「この変更に必要なテストケースを列挙してください」
- 「このコードを読みやすくリファクタしてください」

## 注意点

- 提案をそのまま採用しないこと
- 要件、セキュリティ、例外処理を必ず確認すること
- テストやレビューで妥当性を確かめること

## 章末チェック

- Copilot への依頼文にファイル名と関数名を含められること
- 提案コードのロジックを自分で説明できること
- テストを追加して動作を確認できること
