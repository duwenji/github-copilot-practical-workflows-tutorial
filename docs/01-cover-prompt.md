# ムード適用
カバーはエンジニアリングの精密さとAI支援による開発加速の融合を表現します。Gitブランチツリーが実務フロー（Issue → Branch → PR → Merge → Release）を可視化し、Copilotの紫光（AI sparkle）がワークフロー上で人とAIの協調を演出します。深いネイビーを基調に、マージグリーンとCopilotパープルのコントラストが開発の信頼性と先進性を伝えます。ツールの「機能」ではなく「仕事の流れ」という実践的なコンセプトを視覚で体現します。

# フォント適用
モダンなサンセリフ書体を使用します。タイトルはミディアムボールドで視認性を確保し、サブタイトルはレギュラーウェイトでワークフローのステップ感と実践的な学習の流れを示します。

# コンポジション（縦長キャンバス）

テキストコンポジション:
- タイトルとサブタイトルは上部ゾーン（上部25%）に配置し、Gitブランチグラフのイラストエリアと明確に分離する。
- タイトルゾーンは半透明のダークカード上に配置し、深みのある背景と調和させる。
- テキストは中央揃えのクリーン配置で、ワークフローの整然とした流れを反映する。

ビジュアルコンポジション:
- メインビジュアル（中央60%）: Gitブランチグラフ（main ブランチから feature ブランチが分岐し、PR を経てマージされる流れ）を中央に配置。各コミットノードを丸いドットで表現し、ブランチラインは細い直線で接続する。
- AI Copilot 要素: ブランチグラフ上の複数ポイント（実装・PR説明・レビュー応答）に紫の sparkle アイコンまたはチャットバブルシンボルを重ね、AI が開発フローに溶け込む様子を示す。
- ワークフローアイコン列（下部15%）: Issue（円形バッジ）→ Branch（分岐矢印）→ PR（プルリクエスト矢印）→ Merge（チェックマーク）→ Release（タグアイコン）を小アイコンとして水平に並べ、学習フローを視覚化する。
- 背景: 深いネイビーに微細な六角形グリッドまたはコンステレーションパターンを配置し、GitHubのネットワーク・コラボレーション感を表現するテクスチャとする。不透明度は低めに設定し、主要要素の視認性を損なわない。
- アクセント: release タグ（`v1.0.0` 形式）と GitHub Actions のワークフロー矢印を点在させ、CI/CD・リリース運用の要素を加える。
- デコレーション: `issue` / `PR` / `fork` / `release` などのキーワードラベルを極小テキストとして各要素近傍に配置し、チュートリアルの技術的内容を強調する。
